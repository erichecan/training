# 袋中球童 · 设计与决策记录 (DESIGN)

本文件是整个讨论的结论汇总。部署步骤见 `README.md`，App 本体是 `public/index.html`。

## 1. 产品定位
- 个人高尔夫训练工具，**非商业、要免费、贴合自己的工作流**。
- 现实工作流：赛前 yards book 只给距离 → 去 18birds 逐洞看障碍/果岭 → 自己定每杆策略。
- 目标体验：训练时**拍记分卡 → 自动生成全 18 洞策略 → 逐洞翻看**，再**记录实际表现反哺策略**。

## 2. 核心引擎：球包间距 → 单洞策略
- 球包每支杆两个数：**Total**（落地+滚动，开球用）/ **Carry**（飞行落地，攻果岭定落点用）。
- 开球：在长杆里选一支，既避免留尴尬进攻距离、又尽量留顺手满杆距离；该收杆时提示别上 Driver。
- 攻果岭：按 Total≈本距离选杆，用 Carry 定落点（"打 115 借滚动到 120 旗"就是这么来的）。
- 旗位 前/中/后 = 目标 ±6 码；果岭软硬影响落点滚动。
- 风：本距离做 plays-like 修正——逆风约 +1 码/mph、顺风约 −0.6、侧风只提示瞄风。逐洞设风向。
- Par 3 / 4 / 5 分别处理：Par 5 给"两上 vs 放球道留满杆 wedge"两套；短 Par 4 提示可冲果岭。

## 3. 数据分层与 MVP 取舍
1. **距离层**（记分卡 / yards book，可 OCR）——核心，已做。
2. **障碍层**（球道沙坑/水的位置）——记分卡没有，需手标或球场图。**暂缓**。
3. **果岭层**（坡度/滚动）——需航测数据，最贵。先用"软硬"粗略代替。
- 决策：MVP 只用 **距离 + 风** 跑全 18 洞；障碍/果岭以后再加。

## 4. 计划 + 实际 合并到一页
- 每洞一张卡：**上半 = 计划（目标）**，**下半 = 实际（完成）**。顶部对比"目标 Par X · 实际 Y"。

## 5. 击球记录数据模型
- 每杆一条：**选杆 + 落点 + 方向(可选) + 距离(可选)**；每洞记 **推杆数**。
  - 落点：球道 / 长草 / 球道沙坑 / 果岭边缘 / 果岭 / 果岭沙坑 / 水 / OB / 其它
  - 方向：左 / 短 / 准 / 长 / 右
- 自动统计：杆数(击球+罚杆+推杆)、FIR、GIR(到果岭杆数 ≤ Par−2)、Scrambling 救球、沙坑救球、罚杆。

## 6. 离散度 / 偏向 → 反哺策略（关键闭环）
- 按方向标注算每支杆的**偏左/偏右、偏长/偏短、落点离散 ±码**（取距离半幅）。
- 攒够约 4 杆同杆数据后，自动写进策略提醒：开球瞄哪边、攻果岭离散大"打中心别冲边旗"、偏短"宁大半号"。
- 实测距离汇总 → 一键校准球包 Total/Carry。**打得越多，策略越准。**
- 现状为**定性建议**；要做到**定量落点**（"瞄果岭中心偏左 5 码"）还差两样数据：
  果岭旗位/沙坑相对位置 + 足够多的带方向记录 ——这是下一步重点。

## 7. 存储演进
- Claude artifact 版：`window.storage`（键值，跨会话）。
- 可移植 shim：有 `window.storage` 就用，否则退回 `localStorage`。
- 自托管版（本项目）：Neon 单张 `kv(k, v)` 表；前端 `store` 走 `/api/kv`；离线退回 `localStorage`。
  键沿用 `golf:bag2` / `golf:course2` / `golf:rounds` / `golf:curRound`，前端几乎不用动。

## 8. 部署架构（Cloud Run + Neon）
- **一个 Cloud Run 服务**：发 PWA 静态文件 + `/api/kv`(存 Neon) + `/api/scan`(OCR 代理)。
- Neon 用**带 `-pooler` 的连接串**（Cloud Run 多实例，避免连接耗尽）。
- OCR 的 **Anthropic API key 只在服务端**（Secret Manager 注入），前端永不接触。
  GCP 原生更省事：`/api/scan` 改用 **Vertex AI / Gemini**，靠 Cloud Run 服务账号鉴权，**免 key**。
- PWA：`manifest.json` + `sw.js` + HTTPS(Cloud Run 自带) → 手机"添加到主屏幕"即装。

## 9. 待办 / 下一步（按优先级）
1. **果岭旗位 + 沙坑相对位置标注**，结合离散度给出具体落点建议（把第 6 点做成定量）。
2. 障碍（球道沙坑/水）逐洞快速标注，进策略避让。
3. `/api/scan` 改 Vertex AI 版本（可选，免 key）。
4. 出 PWA 图标 192/512 PNG，放进 `public/`。
5. 多设备/多人 → `kv` 表加 `user_id`，主键改 `(user_id, k)`。
