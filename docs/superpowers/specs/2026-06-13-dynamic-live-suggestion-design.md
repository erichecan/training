# 逐杆实时建议增强(动态策略 · A 方案)

> 日期:2026-06-13
> 范围:`public/index.html` 单文件,纯前端逻辑,无 schema / 无后端 / 无 AI / 无联网

## 背景与目标

App 现有两套打法建议:

1. **18 洞静态策略卡(`computePlan` → `S.plans`)** — 扫码后一次算死,假设站在发球台理想发挥,不随击球变。**本次保持静态,不动。**
2. **逐杆实时建议(`liveSuggest`)** — 已存在但偏弱:在当前展开的那一杆编辑器顶部显示一条 `🎯 按实际剩 X码 → 某杆`,基于上一杆的实际剩余距离(`prev.rem`)用标称距离选杆,附带球位/方向修正。

用户痛点:**实时建议「不够准 / 不够具体」**。

本次目标:在不改触发条件、不改 UI 大结构的前提下,让 `liveSuggest` 这条建议**更准、更具体**。三个增强方向(纯本地逻辑,用户明确不走 AI):

- ① 用你的**真实历史距离**选杆(不是球包标称)
- ② 结合**本轮手感**(今天这场的方向偏差/长短)
- ③ 更**具体的打法指引**(旗位 / 果岭软硬 / 风带来的取舍)

## 改动边界

| 改动 | 内容 |
|------|------|
| 重写 | `liveSuggest(i, j)`(public/index.html:743) |
| 新增 | `pickClubByHistory(rem)` — 历史真实距离选杆 |
| 新增 | `thisRoundSignal(club)` — 本轮(当前 `R`)手感信号 |

| 不碰 | 原因 |
|------|------|
| `computePlan` / `S.plans` 静态策略卡 | 用户要求保持静态 |
| 数据结构 `R.holes[].shots[]` | 现有字段(club/dist/result/dir/rem/pos)已够,无 schema 变更 |
| 后端 `/api/coach` | 纯本地逻辑,不联网不走 AI |
| 触发条件(上一杆有 `rem`、开球第 1 杆不触发) | 用户未把「门槛高」列为痛点,沿用现机制 |
| `live-sg` 容器 / CSS 大结构 | 只丰富内容,不新增 UI 元素 |

**触发条件不变**:仍然是上一杆填了「距旗」或标了 GPS 球位反推出 `rem`,下一杆(第 2 杆起)才显示实时建议。本次只让建议变准变具体,不改它何时出现。

## 复用的现有积木

- `clubHistoryAt(rem, tol)`(public/index.html:934) — 按距离段(默认 ±12% 容差)聚合历史各杆的次数 `n` / 均距 `avg` / 结果分布 `res`。现在只拿来「参考」显示,本次让它**真正参与选杆**。
- `clubTendency(club)`(public/index.html:953) — 某杆全历史的偏左/右、偏长/短、离散度 `spread`、均距。
- `approachShot(...)`(public/index.html:660) — 标称 carry/total 选杆,作为历史数据不足时的**回退**。
- `nd(dir)`、`RESULTS`、`topResLabel` 等工具沿用。

## 增强 ① 真实距离选杆 — `pickClubByHistory(rem)`

逻辑:
- 调 `clubHistoryAt(rem)`,看历史上这个距离用过哪些杆。
- 该距离段历史累计 **≥3 杆** → 用历史选杆,排序口径:**结果优先**(上果岭 / 落点好的比例高者优先),并列时频次(`n`)多者优先。
  - 措辞示例:`🎯 剩 132码 → 你这距离 P 上过 4 次(3 次上果岭),用 P`
- 样本不足(<3)→ 返回 null,`liveSuggest` 回退到现有 `approachShot` 标称选杆,措辞照旧。

「结果好」的判定:`res` 里上果岭类结果(`on` 等正向 key)占比;具体正向 key 以现有 `RESULTS` 定义为准,实现时确认。

## 增强 ② 本轮手感 — `thisRoundSignal(club)`

只看**当前这一轮 `R`** 已记录的杆(区别于看全历史的 `clubTendency`):
- 本轮某杆 **≥2 次**同向偏(如 driver 今天 2 次右)→ `⚠️【本场】这支老右曲,瞄左一点`
- 本轮某杆明显偏短 / 偏长(≥2 次)→ 提示加 / 减半号
- 阈值 **2 次起报**;1 次偶发不报,避免误导。

## 增强 ③ 具体打法指引(并入 `liveSuggest` 措辞)

基于 `rem` + 旗位 `S.holePin[i]` + 果岭软硬 `S.firm` + 风 `S.holeWind[i]`/`S.windMph`,给取舍语言(多为把 `approachShot` / `tendencyNote` 已有信号说得更明确):
- 前旗 → `别冲短旗,瞄果岭中部,落球后旗更安全`
- 后旗 → `后旗可加半号,但别长`
- 够不着果岭(rem > 最长上手杆)→ `够不到,放到剩 ~90 码留个顺手距离`
- 该杆历史离散大(`spread≥8`)→ `这支飘,打中心别冲边旗`
- 很近(rem ≤ 最短杆)→ `留上坡推,别打过旗`

增强 ③ **不依赖历史**,始终可给(新用户也有)。

## 降级链(核心)

**修正建议优先级(增强② vs 全历史):**
```
本轮某杆已有 ≥2 次同向信号  →  优先「本场」修正,措辞标【本场】
        ↓ 不足(如第 1–2 洞,本轮样本 < 2)
退回全历史 clubTendency      →  措辞标【历史】,且仅在历史样本够(现有 dirN≥4 阈值)时才报
        ↓ 都不够(新用户 / 新杆)
不报修正,只给选杆 + 打法细节,不硬凑
```

**选杆主体降级链(增强①):**
```
该距离段历史 ≥3 杆  →  结果优先选杆,措辞带历史依据
        ↓ 不足
退回 approachShot 标称选杆(现有逻辑),措辞照旧
```

## 边界(必须优雅处理,不得崩)

- 新用户**零历史** → 全程走标称回退,行为同现在,只多了增强③的旗位/风打法细节。
- `prev.rem` 为空(没填距旗)→ 同现在,不显示实时建议。
- 本轮第 1 杆(开球)→ 维持现状,`liveSuggest` 仍从第 2 杆起。
- 历史 `dist` 离群值(GPS 误差 400 码)→ `clubHistoryAt` 的 ±12% 距离段容差天然过滤。

## 呈现微调(很轻)

现有 `live-sg` 容器结构不变,内容更丰富:
- 第一行:选杆结论(带历史依据)
- 第二行:打法细节(增强③)
- 底部:最多两条 ⚠️(【本场】/【历史】/球位修正)

不新增 UI 元素,不改 CSS 大结构。

## 验证要点

- 零历史新用户:实时建议不报错,走标称回退 + 打法细节。
- 有历史:同距离段 ≥3 杆时,选杆切到历史结果优先并显示依据。
- 本轮某杆 2 次同向偏:出现【本场】修正,且排在【历史】之前。
- 改 `sw.js` CACHE 版本号(改了 index.html 必须 bump,否则手机端拿不到新版)。
